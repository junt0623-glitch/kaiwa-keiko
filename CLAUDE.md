# CLAUDE.md — 会話稽古（英語・中国語 音声チャット学習ツール）

このファイルはClaude Codeへの引き継ぎ文書です。作業前に必ず全体を読んでください。

## プロジェクト概要

音声チャットで英語・中国語（普通話）の会話を練習する単一HTMLツール。
美術館学芸員である開発者が、館務（作品案内・来館者対応・学会交流）の
語学練習に使う。GitHub Pagesで公開する。

## 絶対に守る設計制約（変更禁止）

1. **index.html 1ファイル完結**。外部ライブラリ・CDN読み込み・ビルド工程・npm・フレームワークは一切使わない
2. デプロイは GitHub Pages（mainブランチ直下の index.html）
3. コード内コメント・UI文言・エラーメッセージはすべて日本語
4. ネット通信は「音声認識（ブラウザ内蔵）」「AI API呼び出し」の2つだけ。読み上げ・UIはオフラインで動くこと
5. APIキーはユーザー各自がブラウザに保存する方式。コードやリポジトリにキーを書かない

## アーキテクチャ（現状）

- **音声入力**: Web Speech API（`webkitSpeechRecognition`）。言語は en-US / zh-CN
- **読み上げ**: `speechSynthesis`。言語一致するvoiceを自動選択、速度スライダー付き
- **ハンズフリーループ**: 🎤ボタン＝セッション開始/終了のトグル。
  聞き取り → 無音で自動送信 → AI応答 → 読み上げ → 約400ms後に聞き取り再開。
  AI読み上げ中はマイクを開かない（自己音声の誤認識防止）。
  エラー時はセッションを自動停止する（無限ループ防止）
- **マイク許可**: `getUserMedia` で取得したストリームをページ表示中は保持し、
  再許可ダイアログを抑止（`ensureMicPermission`）
- **AIバックエンド**: Claude / Gemini のデュアル対応
  - Claude: `claude-sonnet-4-6`。キー有時は `anthropic-dangerous-direct-browser-access` ヘッダ必須
  - Gemini: **モデル名をハードコードしない**。`/v1beta/models` から自動検出し
    flash-lite系を最優先。404が返ったら再検出して1回リトライ。
    `thinkingConfig:{thinkingBudget:0}` で高速化し、400が返るモデルでは自動的に外す
- **応答形式**: AIにはJSONのみを返させる
  `{reply, reading(中国語時はピンイン), translation(日本語訳), feedback(日本語の添削・null可)}`
  パースは `extractJSON`（括弧の深度を数える方式）で、末尾のゴミや余分な `}` に耐える
- **設定保存**: localStorage を try/catch でラップ（`store`）。
  キー: `kk_key`（APIキー）, `kk_provider`, `kk_gemini_model2`（検出済みモデル）

## デザイン

青磁（celadon）を基調とした配色。CSS変数 `--ink / --celadon / --porcelain / --cinnabar` を使用。
見出しは明朝系、本文はシステムサンセリフ。録音中は辰砂色のリップルアニメーション。
`prefers-reduced-motion` に対応済み。新UIもこのトーンを維持すること。

## 既知の注意点

- 音声認識はブラウザの仕様でネット接続必須。`file://` で開くと許可が保存されない
- Geminiのモデル名は頻繁に世代交代するため、自動検出の仕組みを壊さないこと
- iOS Safariでは `speechSynthesis` の初回発声にユーザー操作が必要
- 無音が続くと認識が自動終了→再開を繰り返す（仕様）。再開待ちは400ms
